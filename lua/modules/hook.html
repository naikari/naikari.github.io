Module hook

Lua bindings to manipulate hooks.

Hooks allow you to trigger functions to certain actions like when the player jumps or a pilot dies.

The function an action is hooked to is passed a list of hook parameter arguments, plus an optional argument. The hook parameters (and how many hook parameters there are) vary by the type of action being hooked.

Example usage would be: 

 function player_board(target, arg)
    -- Causes the player to get paid any time they board a ship from
    -- the faction specified by the optional argument.
    if target:faction() == arg then
       player.pay(10000)
    end
 end

 -- Hooks the function, specifying the Pirate faction as the optional
 -- argument.
 hookid = hook.board("player_board", faction.get("Pirate"))

land (funcname[, where[, Argument]])
Hooks the function to the player landing.

Can also be used to hook the various subparts of the landing menu. Possible targets for where are:

  • "land": when landing (default with no parameter)
  • "outfits": when visiting outfitter
  • "shipyard": when visiting shipyard
  • "bar": when visiting bar
  • "mission": when visiting mission computer
  • "commodity": when visiting commodity exchange
  • "equipment": when visiting equipment screen

The hook receives only the optional argument.

Parameters:

  • funcname string Name of function to run when hook is triggered.
  • where string Where to hook the function. (optional)
  • Argument to pass to hook. (optional)

Returns:

    number Hook identifier.

Usage:

  • hook.land( "my_function" ) -- Land calls my_function
  • hook.land( "my_function", "equipment" ) -- Calls my_function at equipment screen

safe (funcname[, arg])
Hook run once at the end of the next frame.

This hook runs once (and only once) at the end of the next frame, regardless of anything that can happen, and removes itself. This is a good way to do possibly breaking stuff like for example player.teleport().

The hook receives only the optional argument.

Parameters:

  • funcname string Name of function to run when hook is triggered.
  • arg Argument to pass to hook. (optional)

Returns:

    number Hook identifier.

See also:

    update
update (funcname[, arg])
Hook run at the end of each frame when the update routine is run.

This hook is similar to hook.safe(), but you have to manually remove it or it continues forever. Also, it is not run while the game is paused or a modal dialog is open, whereas hook.safe() will run even in those cases.

The parameters passed to the function are the current delta-tick (time passed in game) and real delta-tick (independent of game status).

Parameters:

  • funcname string Name of function to run when hook is triggered.
  • arg Argument to pass to hook. (optional)

Returns:

    number Hook identifier.

See also:

    safe

custom (hookname, funcname[, arg])
Custom hook that must be manually triggered.

Custom hooks can be used to cause missions and events to affect each other. Use hook.trigger() to trigger the hook at the appropriate time.

Hook parameters (and number of hook parameters) are chosen by the respective hook.trigger() call(s).

Parameters:

  • hookname string Name to give the hook. This should not overlap with standard names.
  • funcname string Name of function to run when hook is triggered.
  • arg Argument to pass to hook. (optional)

Returns:

    number Hook identifier.

See also:

    trigger
trigger (hookname[, params])
Manually triggers a hook.

This is run deferred (next frame) globally (i.e. it affects all missions and events). Meant mainly to be used with hook.custom(), but can work with other hooks too (if you know what you are doing).

Parameters:

  • hookname string Name of the hook to be run.
  • params nil, number, boolean, string, Pilot, Faction, Planet or Jump Arguments to pass to the hook's parameters. You can pass multiple arguments, but this is limited by HOOK_MAX_PARAM (which is defined in hook.h). (optional)

See also:

    custom
pilot (pilot, type, funcname[, Argument])
Hooks the function to a specific pilot.

You can hook to different actions. Currently hook system supports:

  • "death": triggered when the pilot dies (before marked as dead). Hook parameters are the dying pilot and the pilot that killed them (or nil if not killed by a pilot).
  • "exploded": triggered when the pilot has died and the final explosion has begun. Hook parameter is the exploded pilot.
  • "kill": triggered when the pilot kills another pilot (before marked as dead). Hook parameters are the killing pilot (or nil if not killed by a pilot) and the killed pilot.
  • "boarding": triggered when the pilot boards another pilot (start of boarding). Hook parameters are the boarding pilot and the pilot being boarded.
  • "board": triggered when the pilot is boarded (start of boarding). Hook parameters are the pilot being boarded and the boarding pilot.
  • "disable": triggered when the pilot is disabled (with disable set). Hook parameters are the disabled pilot and the pilot that disabled them (or nil if not disabled by a pilot).
  • "undisable": triggered when the pilot recovers from being disabled. Hook parameter is the recovered pilot.
  • "jump": triggered when the pilot jumps to hyperspace (before they actually jump out). Hook parameters are the jumping pilot and the jump point being jumped through.
  • "hail": triggered when the pilot is hailed by the player. Hook parameter is the pilot being hailed.
  • "land": triggered when the pilot is landing (right when starting land descent). Hook parameters are the landing pilot and the planet being landed on.
  • "attacked": triggered when the pilot is attacked. Hook parameters are the attacked pilot, the attacking pilot (or nil if not attacked by a pilot), and the damage inflicted.
  • "idle": triggered when the pilot becomes idle in manual control. Hook parameter is the idle pilot.
  • "lockon": triggered when the pilot locked on a missile on its target. Hook parameter is the pilot which achieved the lockon.

If you pass nil as pilot, it will set it as a global hook that will trigger for all pilots.

Do not do unsafe things in pilot hooks. This means stuff like player.teleport(). If you have doubts, use a "safe" hook.

Parameters:

  • pilot Pilot or nil Pilot identifier to hook (or nil for all).
  • type string One of the supported hook types.
  • funcname string Name of function to run when hook is triggered.
  • Argument to pass to hook. (optional)

Returns:

    number Hook identifier.
