Module pilot

Lua bindings to interact with pilots.

This will allow you to create and manipulate pilots in-game.

An example would be: 

 p = pilot.add( "Llama", "Miner" ) -- Create a Miner Llama
 p:setFriendly() -- Make it friendly

add (shipname, faction[, source[, pilotname[, parameters]]])
Adds a ship with an AI and faction to the system.

How the "source" argument works (by type of value passed):

  • nil: spawns pilot randomly entering from jump points with presence of their faction or taking off from non-hostile planets
  • planet: pilot takes off from the planet
  • system: jumps pilot in from the system
  • vec2: pilot is created at the position (no jump/takeoff)
  • true: Acts like nil, but does not avoid jump points with no presence

Arguments that can be passed to the "parameters" parameter:

  • "ai" (string): AI to give the pilot. Defaults to the faction's AI.
  • "naked" (boolean): Whether or not to have the pilot spawn without outfits. Defaults to false.
  • "noequip" (boolean): Whether or not to skip the equip script (and use the ship's default outfits). Defaults to false.

Parameters:

  • shipname string Raw (untranslated) name of the ship to add.
  • faction Faction Faction to give the pilot.
  • source System, Planet, vec2 or boolean Where to create the pilot; see above for a complete explanation. (optional)
  • pilotname string Translated name to give the pilot. Defaults to the translated version of shipname. (optional)
  • parameters table Table of extra keyword arguments. See above for supported arguments. (optional)

Returns:

    Pilot The created pilot.

Usage:

  • p = pilot.add("Empire Shark", nil, "Empire") -- Creates a standard Empire Shark.
  • p = pilot.add("Hyena", "Pirate", _("Pirate Hyena")) -- Just adds the pilot (will jump in or take off).
  • p = pilot.add("Llama", "Trader", nil, _("Trader Llama"), {ai="dummy"}) -- Overrides AI with dummy ai.
  • p = pilot.add("Gawain", "Civilian", vec2.new( 1000, 200 )) -- Pilot won't jump in, will just appear.
  • p = pilot.add("Empire Pacifier", "Empire", system.get("Goddard")) -- Have the pilot jump in from the system.
  • p = pilot.add("Goddard", "Goddard", planet.get("Zhiru") , _("Goddard Goddard")) -- Have the pilot take off from a planet.

clear ()
Clears the current system of pilots. Used for epic battles and such.

Pilots which have been set to not clear with pilot.setNoClear(), as well as the player's fighter bay escorts, are exempt and will remain in the system.

Be careful with this function. It will most likely cause issues if multiple missions are in the same system. For this reason, it should only be used in a successfully claimed system.

Note: Clears all global pilot hooks too.

See also:

    setNoClear

Usage:

    pilot.clear()

weapset (p[, id])
Gets the weapset weapon of the pilot.

The slots have the following structure:

  • "name": Raw (untranslated) name of the outfit.
  • "cooldown": [0:1] value indicating if ready to shoot (1 is ready).
  • "charge": [0:1] Charge level of beam weapon (1 is full), or nil if not applicable.
  • "ammo": Raw (untranslated) name of the ammo or nil if not applicable.
  • "max_ammo": Absolute max ammo or nil if not applicable.
  • "left": Absolute ammo left or nil if not applicable.
  • "left_p": [0:1] Relative ammo left or nil if not applicable.
  • "lockon": [0:1] Lock-on percentage for seeker weapons or nil if not applicable.
  • "in_arc": Whether or not the target is in targeting arc or nil if not applicable.
  • "level": Level of the weapon (1 is primary, 2 is secondary, 0 is neither primary nor secondary)).
  • "instant": The instant mode weapon set the weapon is in if applicable, or nil if the weapon is not in an instant mode weapon set.
  • "temp": Temperature of the weapon. The general range is [0:1], but can actually exceed the nominal "maximum" value of 1, so you should take care to make sure that your code can handle values greater than 1 (e.g. by capping the value before putting it somewhere that expects a strict [0:1] range).
  • "type": Raw (untranslated) name of the outfit's type as would be returned by outfit.type().
  • "dtype": Raw (untranslated) name of the weapon's damage type.
  • "track": [0:1] How well the weapon is currently tracking its target.

An example would be: 

 ws_name, ws = p:weapset(true)
 print("Weaponset Name: " .. ws_name)
 for i, w in ipairs(ws) do
    print("Name: " .. w.name)
    print("Cooldown: " .. tostring(cooldown))
    print("Level: " .. tostring(level))
 end

Parameters:

  • p Pilot Pilot to get weapset weapon of.
  • id number or boolean ID of the set to get information of. Set to true to get all equipped weapons. Defaults to the current weapon set. (optional)

Returns:

  1. string The name of the set.
  2. {table,...} A table with each slot's information.

Usage:

  • set_name, slots = p:weapset(true) -- Gets info for all active weapons
  • set_name, slots = p:weapset() -- Get info about the current set
  • set_name, slots = p:weapset(5) -- Get info about the set number 5
weapsetHeat (p[, id])
Gets heat information for a weapon set.

Heat is a 0-2 value that corresponds to three separate ranges:

  • 0: Weapon set isn't overheating and has no penalties.
  • 0-1: Weapon set has reduced accuracy.
  • 1-2: Weapon set has full accuracy penalty plus reduced fire rate.

Parameters:

  • p Pilot Pilot to get weapset weapon of.
  • id number, string or boolean ID of the set to get information of. Defaults to currently active set. Strings identifying special weapon sets can also be used here; see ai.weapset for more information. Set to true to get information of all weapon sets. (optional)

Returns:

  1. number Mean heat.
  2. number Peak heat.

Usage:

  • hmean, hpeak = p:weapsetHeat( true ) -- Gets info for all active weapons
  • hmean, hpeak = p:weapsetHeat() -- Get info about the current set
  • hmean, hpeak = p:weapsetHeat( 5 ) -- Get info about the set number 5
actives (p[, sort=false])
Gets the active outfits and their states of the pilot.

The active outfits have the following structure:

  • "name": Raw (untranslated) name of the activated outfit.
  • "type": Raw (untranslated) name of the outfit's type as would be returned by outfit.type().
  • "temp": [0:1] The heat of the outfit's slot, or nil if not applicable.
  • "weapset": The first weapon set that the outfit appears in, if any, or nil if not applicable.
  • "state": State of the outfit, which can be "off", "on", or "cooldown".
  • "cooldown": [0:1] Cooldown percentage remaining (0 = just finished cooling down, 1 = just started cooling down), or nil if not applicable.

An example would be: 

 act_outfits = p:actives()
 print( "Weapnset Name: " .. ws_name )
 for i, o in ipairs(act_outfits) do
    print( "Name: " .. o.name )
    print( "State: " .. o.state )
 end

Parameters:

  • p Pilot Pilot to get active outfits of.
  • sort boolean Whether or not to sort the outfits. If this is set to true, outfits are sorted so that outfits with a lower weapset are first, and outfits with no weapset are last. (default false)

Returns:

    {table,...} A table with each active outfit's information.

Usage:

    act_outfits = p:actives() -- Gets the table of active outfits

flags (p)
Gets the pilot's flags.

Valid flags are:

  • "carried": Pilot comes from a fighter bay.
  • "hyperspace": Pilot is performing a jump. "hyperspace_end" and "localjump" can co-occur with this. If both "hyperspace_end" and "localjump" are false, "hyperspace" means that the pilot is entering hyperspace (exiting the system).
  • "hyperspace_end": pilot is exiting hyperspace (entering the system).
  • "localjump": Pilot is performing a local jump.
  • "hailing": Pilot is hailing the player.
  • "boardable": Pilot is boardable while active.
  • "boarded": Pilot has been boarded already.
  • "noboard": Pilot can't be boarded.
  • "boarding": Pilot is currently boarding its target.
  • "nodisable": Pilot can't be disabled.
  • "disabled": Pilot is disabled.
  • "disabled_perm": Pilot is permanently disabled.
  • "nodeath": Pilot cannot die, will stay at 1 armor.
  • "invincible": Pilot cannot be hit.
  • "invinc_player": Pilot cannot be hit by the player.
  • "hostile": Pilot is hostile toward the player.
  • "friendly": Pilot is friendly toward the player.
  • "combat": Pilot is engaged in combat.
  • "bribed": Pilot has been bribed.
  • "distressed": Pilot has distressed already.
  • "landing": Pilot is currently landing.
  • "takingoff": Pilot is currently taking off.
  • "norender": Pilot does not get rendered.
  • "visplayer": Pilot is always visible to the player.
  • "visible": Pilot is always visible.
  • "invisible": Pilot doesn't appear on the radar and cannot be targeted, but can still do stuff and is rendered.
  • "hide": Pilot is not updated or rendered and cannot be interacted with.
  • "hilight": Pilot is hilighted on the map.
  • "afterburner": Pilot has their afterburner activated.
  • "refueling": Pilot is refueling another pilot.
  • "cooldown": Pilot is in active cooldown mode.
  • "manualcontrol": Pilot is under manual control.
  • "nojump": Pilot cannot jump.
  • "noland": Pilot cannot land.
  • "persist": Pilot persists when the player jumps.
  • "noclear": Pilot isn't removed by pilot.clear().

Parameters:

  • p Pilot Pilot to get flags of.
